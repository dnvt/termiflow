# Maestro Pulse Guide

`/maestro:pulse` is for recurring research collection, not one-off deep dives.

## What Pulse Is For

- watch a stable set of themes over time
- collect new external signals on a schedule
- produce lightweight findings that can later be ingested or acted on

## Good Pulse Output

Each pulse run should record:

- what was searched
- what changed since the last run
- the strongest findings
- which findings deserve ingest, planning, or no action

Use `/maestro:research` when the question is specific and decision-bound.
Use `/maestro:ingest --pulse` when pulse findings need structured triage.
