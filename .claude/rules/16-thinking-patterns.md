# Thinking Patterns

## Core Principle

Think with the user and the evidence, not ahead of either. The
goal is to generate better options, isolate uncertainty, and move
cleanly from ambiguity to action.

## Five Thinking Modes

Name the mode when the work is non-trivial so the workflow stays
deliberate.

### 1. Diverge

- Generate multiple plausible approaches before judging them
- Push past the first obvious answer
- Use constraint shifts, reversals, and analogies to widen the
  search space

### 2. Converge

- Apply explicit criteria
- Force-rank options instead of preserving false ties
- Say what is being cut and why

### 3. Challenge

- Steel-man the strongest opposing view first
- Ask what would have to be true for the leading approach to fail
- Separate taste disagreements from real failure modes

### 4. Synthesize

- Connect ideas across threads, artifacts, and prior decisions
- Name contradictions instead of smoothing them over
- Produce one coherent framing from scattered evidence

### 5. Structure

- Turn fuzzy thinking into clear concepts with boundaries
- Prefer simple lists, matrices, and trade-off tables over ornate
  frameworks
- If the framework is harder to explain than the idea, remove it

## Usage Cues

Use these modes explicitly when:

- scope or framing is unclear
- the user has too many options
- a leading idea needs stress-testing
- multiple threads need one coherent framing
- fuzzy material needs simple structure
